Hot Place Tracker

OVERVIEW
Hot Place Tracker is a mobile web application that allows users to save and track the locations around the city. It allows users to rate the travel from the starting point to destination. In this project, I ideated the concept, designed work flow, and created an interactive prototype of a live web application.

PRODUCT OVERVIEW
App Concept
Hot Spots Tracker allows user to track the location of cool /hipster places in SF. These places are not a typical tourist locations, but unique locations where the user find it cool to visit. This helps user to keep track of their choice of the city’s most hot spots through uploading pictures and recording their memories of those places.

FINAL PRODUCT
Sign in/ Login
Front-end: Created layout with HTML and CSS and made in responsive to different screen sizes.

Back-end: Implemented the login feature with PHP server and MySQL database hosted by GoDaddy.
Recent tab (map view)
Front-end: Bring Google Maps API to put map view as a background. Put image thumbnails to preview locations.

Back-end: Fetches places data from the MySQL database.
List tab (add location)
Front-end: Used flex to create grid view. Added drop shadows in the upper nav (fixed).

Back-end: Database can be added with using the “Add the place” form.
Location detail
Front-end: Location detail on the bottom popover with the map view also being visible.

Back-end: The database information can be updated when users edit the content.
Comment about the route
Front-end: Pin the starting point and give review to the travel route.

Back-end: Post new starting points to the destination in the database and the trip comments.
Profile
Front-end: Used flex to create the layout. Created pop over for editing profile information.

Back-end: User database can be updated using the popover form. Photo can be uploaded directly from users’ computer.
