Catchup

OVERVIEW
Catchup is a job application manager that helps users to check application status, organize inbox, and arrange interview schedules.

I participated in the end-to-end process from idea planning and research to creating visual component library for the overall looks and feel of the product.

PROJECT OVERVIEW
Problem Statement
Each company has different deadlines, requirements and application processes. We found out that students submit multiple applications a day so it's hard to keep track of all of them. Students usually organize their applications using Notes, Excel, or just keep in general email inbox, which make them difficult to follow up on each application status.
Concept Video
RESEARCH
Survey & Interview
Key findings
• 70% responded they apply more than 10 job applications a day.
• How they manage applications: Google spreadsheet, notes, check email frequently.
• Managing multiple applications, tracking / following up status, and managing job related emails are the most painful parts of the process.
Competitive analysis
1. Monday.com
2. Gmelius
Key takeaways
• Visualization: Provide clear visual cues for different application status so that users can easily distinguish their status.
• Organization: Help users to organize great number of job applications.
• Ease: Easy to manage their schedules and job applications with few clicks.

FINAL PRODUCT
Link to clickable prototype
Inbox
What we offer
• Search by keywords: With this function, we wanted to help our user to search with specific keywords that they are looking for. This can help user to easily find emails they are looking for among the pool of many other emails.
• Organize by company: We provided square blocks of email container where users can easily distinguish emails from particular companies.
Status
What we offer
• Quick filter: This function helps user to easily filter different applications in particular status that users want to see.
• Application progress card: The card contains information about each application and visualizes current progress.
• Add new applications: Users can also add new application in case the application is not automatically added to the app after they submit their job application.
Schedule
What we offer
• Weekly timeline: This function allows users to know their weekly timeline for the upcoming interviews.
• Interview schedule detail: This section shows more detail about the upcoming interveiws. The information includes company, date / time, type of interview, and information about the interviewer.

CONCLUSION
Reflections
What worked well
In this project, I worked with other designers to create two applications (desktop and watch). This required multiple review sessions and design iterations to have everyone in a same page. Since we all had different visual styles, I decided to create a single design systems to have all our screens to have a consistent look. This helped our work flow to be more efficient and have visual consistency across all screens.
What should have been improved
Since we worked on this large scope project in a short period of time, we didn't have much time to do concept testing. To overcome this, we synced up as much as we could to review different design explorations and share feedback. If we have more time, we want to do more user testing. Also, we want to narrow down our project scope and dive deeper on one particular feature.
